# peptide-discover

A computational pipeline for designing novel peptides against a target protein. Combines generative AI, molecular docking, and physicochemical safety screening into a single CLI tool that runs on consumer hardware.

Not a research paper. Not a clinical tool. A working piece of software that takes a UniProt ID and gives you back ranked peptide candidates.

## What it does

```
UniProt ID or PDB file
        ↓
[1] Fetch sequence + AlphaFold structure
        ↓
[2] Generate novel peptide candidates (PepMLM / ESM-2 650M)
        ↓
[3] Dock candidates against target (AutoDock Vina)
        ↓
[4] Score for toxicity, solubility, BBB penetration, hemolysis risk
        ↓
[5] Rank and export (CSV + JSON)
```

## Install

Requires Python 3.10+, ~8GB GPU VRAM (for PepMLM), and CPU for docking.

```bash
pip install -e ".[dev,pepmlm]"
```

AutoDock Vina binary is bundled at `bin/vina.exe` (Windows). For other platforms, download from [ccsb-scripps/AutoDock-Vina](https://github.com/ccsb-scripps/AutoDock-Vina/releases) and place it at `bin/vina`.

## Quick start

```bash
# Full pipeline against a target
peptide-discover run pipeline Q92847 --track muscle --candidates 20 --length 8

# Just generate candidates, skip docking
peptide-discover run pipeline Q92847 --candidates 30 --skip-docking

# Run individual stages
peptide-discover target fetch Q16620
peptide-discover generate pepmlm Q16620 --candidates 10
```

## What it's NOT

- **Not a drug discovery tool.** These are computational predictions. Real drug discovery requires wet lab validation, animal studies, clinical trials.
- **Not peer-reviewed science.** The scoring functions and thresholds are based on published methods, but this pipeline itself has not been validated against experimental results.
- **Not a complete safety screen.** The local physicochemical screening catches obvious red flags (cysteine-rich toxins, insoluble peptides) but cannot predict rare or complex toxicity.
- **Not a replacement for specialized tools.** If you're doing serious peptide design, use proper tools (RFdiffusion, AlphaFold3, PeptiVerse) with proper hardware (24GB+ VRAM).

## What the data tells you

Binding scores are in kcal/mol, more negative = stronger predicted binding. For context:

| Score range | Interpretation |
|---|---|
| -10 or better | Strong binder (rare) |
| -8 to -9 | Good binder |
| -6 to -8 | Moderate |
| -4 to -6 | Weak |
| > -4 | Poor / non-binder |

**Baseline reference** (from `experiments/baseline_known_peptides.py`):

| Known peptide | Target | Score |
|---|---|---|
| Ipamorelin | GHSR | -8.33 |
| Magainin-2 | LptD | -7.73 |
| Magainin-2 | BamA | -7.02 |
| Selank | AGT | -6.07 |

Candidates generated by this pipeline typically land in the -6 to -9 range against membrane protein targets.

## Design decisions

- **No new models.** This is orchestration, not research. Every ML component (PepMLM, Vina, ESM-2) is existing open-source work.
- **Simple ranking.** Candidates are ranked by docking affinity directly. No learned meta-scores or embedding-based re-ranking.
- **Local screening.** Safety prediction uses physicochemical properties computed from sequence. Slower-and-more-accurate ML tools (ToxinPred3, B3Pred) were tried but have broken dependencies or unreliable web APIs.
- **Runs on consumer hardware.** 8GB GPU is enough. No HPC required.
